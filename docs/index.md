# Cloudinary Provisioning Provider

The Cloudinary Provisioning provider allows you to manage account-level Cloudinary resources
(product environments, access keys, and custom policies) using Terraform or OpenTofu.

## Authentication

The provider supports two authentication methods.

### Account URL (recommended)

Set the `CLOUDINARY_ACCOUNT_URL` environment variable:

```
account://API_KEY:API_SECRET@ACCOUNT_ID
```

### Explicit credentials

Set the `account_id`, `api_key`, and `api_secret` attributes or their environment variable
equivalents (`CLOUDINARY_ACCOUNT_ID`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET`).

## Example Usage

```terraform
# Using CLOUDINARY_ACCOUNT_URL environment variable (recommended)
# export CLOUDINARY_ACCOUNT_URL="account://API_KEY:API_SECRET@ACCOUNT_ID"
provider "cloudinaryprovisioning" {}

# Or using explicit credentials
provider "cloudinaryprovisioning" {
  account_id = "your-account-id"
  api_key    = "your-api-key"
  api_secret = "your-api-secret"
}
```

## Schema

### Optional

- `account_id` (String) Cloudinary account ID. Falls back to the CLOUDINARY_ACCOUNT_ID environment variable.
- `account_url` (String, Sensitive) Cloudinary account URL in the format account://API_KEY:API_SECRET@ACCOUNT_ID. When set, takes precedence over account_id, api_key, and api_secret. Falls back to the CLOUDINARY_ACCOUNT_URL environment variable.
- `api_key` (String) Cloudinary Provisioning API key. Falls back to the CLOUDINARY_API_KEY environment variable.
- `api_secret` (String, Sensitive) Cloudinary Provisioning API secret. Falls back to the CLOUDINARY_API_SECRET environment variable.
